# 前提（Assumptions）
各前提を1行で: 内容 · 根拠 · 検証状態（未検証 / 検証中 / 検証済）· 検証方法 · 失敗時の影響
- 未検証/検証中の前提は risks.md にもリスクとして反映する。
- 検証は Universal `/risk-review` と連携（assumption → risk への昇格を追跡）。

- 利用は個人・家族用、一般公開やデータ収集はしない · 利用者本人の方針 · 検証済 · 本スキル実行時に確認 · 失敗時=リスク/ゲート設定が過小になる
- 進捗保存は `localStorage` で十分（端末固定・同期不要）· 個人用途 · 未検証 · 実機で日跨ぎ・端末変更時の挙動確認 · 失敗時=進捗消失で子どものモチベ低下
- 問題データ（漢字の読み/配当・計算の答え・時刻判定）は正確 · 手作業作成 · 未検証 · 第三者レビュー or 一覧突合 · 失敗時=誤学習（教育アプリとして致命的）
- 子どもが一人で操作できる UI 可読性・操作性がある · ひらがな主体の設計 · 未検証 · 実ユーザー（子ども）での試用観察 · 失敗時=自力で進められず目的未達
- Next.js 16 / React 19 の現行構成で Vercel に問題なくデプロイできる · 既存ビルド成功 · 検証中 · `npm run build` + Vercel デプロイ確認 · 失敗時=公開/共有が滞る
