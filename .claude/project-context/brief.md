# Project Brief

## プロジェクト目的
小学1年生が「こくご（漢字）・さんすう（計算）・とけい（時計）」を、自宅で一人でも楽しく繰り返し練習できる学習ドリル Web アプリを提供する。

## 想定ユーザー
- 主：小学1年生の子ども（一人で操作）
- 副：保護者（端末で開いて子どもに渡す。個人・家族用途）

## 解決する課題
- 紙のドリルは採点・ランダム出題・繰り返しが手間。
- 子どもが自分で進められる、即時フィードバック付きの反復練習手段がほしい。

## MVP方針
- 3教科の最小ドリル（こくご：漢字80字 / さんすう：たし算・ひき算ランダム / とけい：時刻読み）。
- バックエンド・DB・ログインなし。進捗は `localStorage` に日付単位で保持。
- 静的問題データ（`src/data`）＋シンプルな GET API（`/api/questions`）。
- 印刷ページ（`/print`）でオフライン練習も補助。
- ホスティングは Vercel。

## 採用案
（Existing Project Mode — 既存スタックを踏襲。新規方向選定なし）
Next.js 16 App Router + React 19 + TypeScript + Tailwind 4、サーバレス/静的中心の構成。

## 採用理由
- 既に動作する MVP が存在し、スタックが確立済み。
- 個人・家族用途で低リスク・低運用コスト。Vercel デプロイと相性が良い。

## 採用しなかった案
- バックエンドDB＋アカウント管理による進捗同期。

## 採用しなかった理由
- 個人・家族用途では `localStorage` で十分。DB/認証は運用負荷とプライバシーリスク（子ども向け）を増やすだけで MVP に不要。将来サービス化する場合に再検討（`decisions/adr/0001`）。
