# ADR-0001: 初期方向性

- 日付: 2026-06-02
- 状態: Accepted
- 決定者: keisuke

## Context（背景）
既存の小学1年生向け学習ドリル Web アプリ（こくご/さんすう/とけい）に対し、Universal Claude OS のプロジェクト特化レイヤ（`.claude`）を最小構成で生成する。スタックは既に確立（Next.js 16 App Router + React 19 + TypeScript + Tailwind 4、サーバDBなし、進捗は localStorage、Vercel デプロイ）。用途は本人確認のうえ「個人・家族用」。

## 選択肢（A/B/C）
### A案: 現状スタックを踏襲し、localStorage で進捗保持（バックエンドなし）
- 概要: 既存構成のまま最小特化レイヤのみ追加。
- 利点: 運用コスト最小、低リスク、子ども向けでデータ収集なし＝プライバシー懸念が小さい。
- 欠点: 端末間で進捗が同期されない。
- リスク: 進捗消失（影響: 低）。

### B案: バックエンドDB＋アカウントで進捗同期
- 概要: ログイン・サーバ保存を追加。
- 利点: マルチデバイス・分析が可能。
- 欠点: 認証/DB の運用負荷、子ども向け個人情報規制（COPPA 等）の対応コスト。
- リスク: コンプライアンス・セキュリティ（影響: 高）。

### C案: 静的サイト＋印刷中心（インタラクティブ最小）
- 概要: 練習は紙印刷主体。
- 利点: 実装極小。
- 欠点: 即時フィードバックという中核価値を失う。
- リスク: プロダクト価値低下。

## Decision（採用案と理由・不採用理由）
**A案を採用。** 個人・家族用途では localStorage で十分であり、低リスク・低運用コストで MVP の中核価値（即時フィードバック付き反復練習）を満たす。B案は現段階では過剰で、子ども向けデータ収集に伴う規制・セキュリティリスクを不必要に背負う。C案は中核価値を損なうため不採用（印刷は補助機能 `/print` として併存）。

## Consequences（影響・ロールバック条件・再評価日）
- 影響: 品質ゲートは Low（lint・build）。脅威モデル/セキュリティレビューは通常不要。問題データの正確性とテスト不足が主リスク。
- ロールバック/再評価条件: 一般公開やアカウント・進捗同期・データ収集を導入する場合は本 ADR を見直し、Risk Level を Medium 以上へ引き上げ、`/threat-model` と子ども向け規制（`risks.md` Legal）を再評価する。
- 再評価日: 方針変更時、または 2026-12-01。
